# MTG Community Sentiment Analysis: Reddit & Discord
**User Research for Phased Delivery Validation**

## Executive Summary

The MTG community strongly gravitates toward **Moxfield for deck building** but acknowledges **no single app solves everything**. Users express frustration with **multi-app workflows** and consistently request **better collection-deck integration**, **faster card input methods**, and **physical storage tracking**. The community shows high interest in **AI-powered features** that reduce manual work, particularly for collection management.

## Deck Building Sentiment

### Community Consensus: "Moxfield Has Won"

> *"When someone posts a deck link on Reddit or Discord, it's almost always Moxfield"*

**Why Moxfield Dominates:**
- **Fast editor**: Responsive, clean interface
- **Real-time updates**: Mana curve updates as you build
- **Painless sharing**: Easy links for community discussion
- **Community adoption**: Network effects from widespread use

**Archidekt's Niche Appeal:**
- **Auto-categorization**: Automatically sorts cards into functional categories (Removal, Ramp, Card Draw)
- **Bracket system**: Comprehensive deck power level scoring
- **Collection integration**: Better collection-to-deck workflow than competitors

**TappedOut's Community Legacy:**
- **Active forums**: Real engagement in deck comments and discussion
- **Rules discussion**: Community-driven rules questions and answers
- **Custom design**: Platform for custom card design discussions

### Pain Points in Current Tools

**Universal Frustrations:**
1. **No single solution**: "Most serious MTG players end up using a combination"
2. **Workflow fragmentation**: Forced to switch between multiple apps
3. **Collection disconnect**: Deck builders don't integrate well with collection trackers
4. **Manual overhead**: Too much clicking and manual entry

## Collection Management Community Insights

### Physical Collection Challenges

**Storage and Organization:**
- *"As collections grow, keeping everything organized can feel overwhelming"*
- Users struggle with physical location tracking: "Where did I put that card?"
- No effective solution for mapping digital inventory to physical storage

**Large Collection Management:**
- *"Managing large MTG collections can become a hassle as they grow"*
- Digital tools become tedious for collections over 1,000+ cards
- Users want faster input methods as "every extra second per card adds up"

### Digital Tool Limitations

**Input Method Frustrations:**
- *"Electronic inventory management was tedious to input and quickly became out of sync"*
- Manual card entry requires too many steps: name → edition → condition → quantity
- **High Demand**: *"Users wish there were collection managers that could identify cards via camera"*

**App Integration Issues:**
- Collection managers don't track which cards are in active decks vs available
- Poor synchronization between different collection tracking tools
- Mobile app limitations: many tools lack mobile versions or have poor mobile UX

**Scanning Technology Gaps:**
- *"Some applications had scanning issues where they wouldn't recognize cards"*
- Users can't easily note condition, location, or custom details during scanning
- Existing scanning requires perfect lighting and single-card methodology

## Feature Wishlist from Community Discussions

### Highest Priority Requests

**1. Unified Workflow Integration**
- Single app for collection + deck building + trading
- Real-time sync between what you own and what decks need
- Collection availability awareness in deck builder

**2. Advanced Scanning Technology**
- **Multi-card scanning**: Ability to scan binder pages or multiple cards at once
- **Better recognition**: Reliable scanning that works in various lighting conditions
- **Flexible input**: Easy correction when scanning fails

**3. Physical-Digital Bridge**
- Track physical storage locations (which box, binder page, sleeve)
- QR codes or labels to link physical storage to digital inventory
- "Where is this card?" search functionality

**4. Mobile-First Experience**
- Full-featured mobile apps, not just web responsive design
- Offline capability for collection browsing and deck building
- Camera-based features optimized for mobile workflow

### Secondary Requests

**Accessibility Improvements:**
- Text size options for visually impaired players
- High contrast modes
- Alternative input methods

**Social and Community Features:**
- Better deck sharing and collaboration tools
- Community challenges and events
- Local game store integration

**Trading and Marketplace Integration:**
- Automated trade opportunity discovery
- Direct integration with major marketplaces
- Price tracking and alerts

## AI Feature Interest Analysis

### High-Interest AI Applications

**1. Bulk Card Recognition**
- **Community Need**: "Every extra second per card adds up significantly"
- **Desired Capability**: Scan 9-18 cards simultaneously from binder pages
- **Value Proposition**: Eliminate tedious one-by-one card entry

**2. Storage Location Intelligence**
- **Community Need**: "Where did I put that card?" is universal frustration
- **Desired Capability**: AI-powered storage suggestions and tracking
- **Value Proposition**: End the physical card hunt

**3. Trade Optimization**
- **Community Need**: Manual trade hunting is time-consuming
- **Desired Capability**: AI scout for profitable trade opportunities
- **Value Proposition**: Maximize collection value with minimal effort

### Community Readiness for AI

**Positive Indicators:**
- Frustrated with current manual processes
- Willing to try new technology for time savings
- Active discussion about automation possibilities

**Potential Concerns:**
- Privacy around collection data
- Accuracy of AI recommendations
- Cost vs. value of premium AI features

## Competitive Switching Behavior

### What Drives Platform Switching

**Switching Accelerators:**
- Significantly better mobile experience
- Unified collection + deck building workflow
- Reliable scanning technology that "just works"
- AI features that save genuine time

**Switching Barriers:**
- **Data lock-in**: "All my decks/data are there"
- **Community network effects**: "Friends/community use it"
- **Interface familiarity**: Comfort with existing workflows
- **Free tier limitations**: Cost-sensitive user base

### Platform Loyalty Factors

**Strong Loyalty Indicators:**
- Established deck collections (high switching cost)
- Community integration (shared decks, discussions)
- Workflow muscle memory

**Weak Loyalty Indicators:**
- Dissatisfaction with multi-app workflow
- Frustration with scanning reliability
- Desire for better collection integration

## Community Sentiment Summary

### Positive Reception Potential

The community shows **high receptivity** to a unified MTG app that addresses current pain points:

1. **Unified Experience**: Strong demand for single-app solution
2. **Better Mobile**: Current apps lag on mobile functionality
3. **AI Automation**: High interest in reducing manual work
4. **Physical Integration**: Unique opportunity with storage tracking

### Critical Success Factors

**Must-Haves:**
- Deck building experience equal to or better than Moxfield
- Scanning that's more reliable than ManaBox/Delver Lens
- Mobile-first design with full feature parity
- Seamless collection-to-deck workflow

**Nice-to-Haves:**
- Community features comparable to TappedOut
- Advanced analytics like Archidekt's bracket system
- Integration with major trading platforms
- Accessibility features for broader user base
